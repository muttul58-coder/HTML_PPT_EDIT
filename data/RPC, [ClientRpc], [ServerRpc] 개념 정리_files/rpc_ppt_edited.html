CONCEPT
RPC란 무엇인가?
Remote Procedure Call — 네트워크 너머의 다른 머신에 있는 함수를 로컬에서 호출하듯 실행하는 메커니즘입니다.
DEFINITION

RPC(Remote Procedure Call)는 클라이언트 ↔ 서버 간 함수 호출을 가능하게 하는 Netcode의 핵심 통신 방식입니다.

NetworkVariable이 상태(State)를 동기화한다면,
RPC는 이벤트(Event)를 전달합니다.

NGO의 두 가지 RPC 어트리뷰트
[ServerRpc]
클라이언트 → 서버 방향으로 호출.
클라이언트가 서버에 요청을 보낼 때 사용.
[ClientRpc]
서버 → 클라이언트 방향으로 호출.
서버가 모든(또는 특정) 클라이언트에 알릴 때 사용.
NetworkVariable vs RPC
항목NetworkVariableRPC
목적상태 동기화이벤트 전달
지속성✔ 지속 (값 유지)✘ 일회성
새 참가자✔ 자동 수신✘ 미수신
예시HP, 점수, 위치공격, 스킬, 채팅
💡 핵심 규칙
RPC를 사용하려면 해당 클래스가 반드시 NetworkBehaviour를 상속해야 하며, 오브젝트가 네트워크에 스폰(Spawn)되어 있어야 합니다.
⚠ 메서드 명명 규칙
NGO 1.x 기준으로 메서드 이름이 반드시 ServerRpc 또는 ClientRpc로 끝나야 합니다.
예) AttackServerRpc(), ShowEffectClientRpc()
02 / 07
HOW IT WORKS
동작 원리 — 호출 흐름
RPC는 직접 함수를 실행하지 않고, 네트워크 패킷으로 직렬화되어 목적지에서 역직렬화 후 실행됩니다.
[SERVERRPC] 호출 흐름
🖥️ CLIENT — AttackServerRpc() 호출
↓ 네트워크 패킷으로 직렬화
📦 패킷 전송 (파라미터 포함)
↓ 역직렬화 후 실행
👑 SERVER — AttackServerRpc() 실행 ✔
클라이언트에서는 실행되지 않음
호출한 클라이언트 본인도 실행 안 됨 (기본값). RequireOwnership = false 설정으로 소유권 검사 해제 가능.
[CLIENTRPC] 호출 흐름
👑 SERVER — ShowEffectClientRpc() 호출
↓ 네트워크 패킷으로 직렬화
📦 패킷 전송 (모든 클라이언트)
↓ 역직렬화 후 실행
🖥️ CLIENT A 실행 ✔
🖥️ CLIENT B 실행 ✔
서버에서는 실행되지 않음
기본적으로 모든 클라이언트에 전송. ClientRpcParams로 특정 클라이언트만 지정 가능.
03 / 07
[SERVERRPC]
[ServerRpc] 상세 — 클라이언트 → 서버
클라이언트가 서버에 행동을 요청할 때 사용합니다. 서버에서만 실행되며, 게임 로직의 권위는 항상 서버가 갖습니다.
기본 사용법
public class Player : NetworkBehaviour
{
    // 클라이언트에서 호출 → 서버에서 실행
    [ServerRpc]
    private void AttackServerRpc(int damage)
    {
        // 여기는 서버에서만 실행됨
        ApplyDamage(damage);
    }

    private void Update()
    {
        if (!IsOwner) return;
        if (Input.GetKeyDown(KeyCode.Space))
            AttackServerRpc(10);
    }
}
RequireOwnership 옵션
기본값 true — 오브젝트 소유자만 ServerRpc를 호출할 수 있습니다.
false로 설정하면 모든 클라이언트가 호출 가능합니다. 
// 소유권 검사 없이 누구나 호출 가능
[ServerRpc(RequireOwnership = false)]
private void ChatServerRpc(string msg) { }
핵심 규칙
1

메서드 이름은 ServerRpc로 끝나야 함

AttackServerRpc(), MoveServerRpc() 등 접미사 필수

2

반환값 없음 (void만 허용)

결과를 받으려면 ClientRpc 또는 NetworkVariable을 함께 사용

3

서버에서 호출 시 즉시 로컬 실행

서버가 직접 호출하면 네트워크 전송 없이 즉시 실행됨

!

클라이언트에서 호출해도 클라이언트 본인은 실행 안 됨

서버만 실행 — 클라이언트 쪽에서 예측 동작이 필요하면 별도 처리 필요

04 / 07
[CLIENTRPC]
[ClientRpc] 상세 — 서버 → 클라이언트
서버가 클라이언트에게 이벤트를 전달할 때 사용합니다. 기본적으로 모든 클라이언트에 전송되며, 대상을 필터링할 수 있습니다.
기본 사용법 + ClientRpcParams
// 모든 클라이언트에게 전송 (기본)
[ClientRpc]
private void ShowEffectClientRpc(Vector3 pos)
{
    SpawnParticle(pos);
}

// 특정 클라이언트에게만 전송
[ClientRpc]
private void NotifyClientRpc(
    string msg,
    ClientRpcParams rpcParams = default)
{
    Debug.Log($"수신: {msg}");
}

// 서버에서 특정 클라이언트만 지정해서 호출
private void SendToPlayer(ulong clientId)
{
    ClientRpcParams p = new() {
        Send = new() {
            TargetClientIds = new[] { clientId }
        }
    };
    NotifyClientRpc("당신만 받음", p);
}
ClientRpcParams — 전송 대상 제어
ClientRpcParams 구조체의 Send.TargetClientIds에
대상 클라이언트 ID 배열을 지정합니다.
이 파라미터는 반드시 마지막 인자여야 하며,
실제 직렬화 대상에서 제외됩니다.
핵심 규칙
1

메서드 이름은 ClientRpc로 끝나야 함

ShowEffectClientRpc(), UpdateUIClientRpc() 등 접미사 필수

2

서버에서만 호출 가능

클라이언트에서 ClientRpc를 호출하면 무시됨 (경고 없이 실행 안 됨)

3

Host는 서버+클라이언트 동시 실행

Host 모드에서는 네트워크 전송 없이 즉시 로컬 실행됨

!

ClientRpcParams는 직렬화되지 않음

파라미터로 전달되지만 실제 패킷에는 포함되지 않는 메타 정보

PARAMS & LIMITS
파라미터 전달 & 제한 사항
RPC에 전달 가능한 타입과 불가능한 타입, 그리고 알아야 할 제약사항을 정리합니다.
전달 가능 / 불가능 타입
✔ 가능한 타입
int, float, bool
string
Vector2/3/4

✘ 불가능한 타입

GameObject 전달 방법
GameObject 대신 NetworkObjectReference를 사용. 수신 측에서 .TryGet(out NetworkObject no)로 꺼냄.
INetworkSerializable — 커스텀 타입 전달
public struct PlayerStats : INetworkSerializable
{
    public int Hp;
    public float Speed;

    public void NetworkSerialize<T>(
        BufferSerializer<T> s)
        where T : IReaderWriter
    {
        s.SerializeValue(ref Hp);
        s.SerializeValue(ref Speed);
    }
}
// RPC에서 사용
[ClientRpc]
void UpdateStatsClientRpc(PlayerStats stats) { }
⚠ 주요 제한 사항
• RPC는 스폰된 NetworkObject에서만 호출 가능
• 반환값(return) 없음 — void만 허용
• 너무 큰 데이터는 MTU 초과로 분할 전송됨
• 매 프레임 대량 호출 시 네트워크 부하 주의

COMPARISON
ServerRpc vs ClientRpc 비교
두 어트리뷰트의 핵심 차이를 한눈에 비교하고, 각각 어떤 상황에 사용해야 하는지 정리합니다.
항목 [ServerRpc] [ClientRpc]
호출 방향클라이언트 → 서버서버 → 클라이언트(들)
호출 주체클라이언트 (소유자 또는 전체)서버만 가능
실행 위치서버에서만 실행지정된 클라이언트에서 실행
기본 대상서버 1곳모든 클라이언트
소유권 옵션RequireOwnership (기본 true)해당 없음
대상 제어항상 서버ClientRpcParams.TargetClientIds
Host 동작즉시 로컬 실행즉시 로컬 실행
대표 용도공격, 아이템 사용, 채팅 요청이펙트, UI 갱신, 결과 알림
ServerRpc 대표 패턴
클라이언트 입력 → ServerRpc → 서버 검증 → ClientRpc로 결과 전파
예) 공격 요청 → 피해 계산 → 체력 변경 알림
ClientRpc 대표 패턴
서버 이벤트 발생 → ClientRpc → 모든 클라이언트 동시 반응
예) 보스 등장 연출, 게임 종료 화면 표시
황금 조합 패턴
ServerRpc(요청) + NetworkVariable(상태) + ClientRpc(이벤트)
셋을 역할에 맞게 분리하면 가장 견고한 구조가 됩니다.
07 / 07
PITFALLS
자주 하는 실수 & 주의 사항
RPC 사용 시 초보자가 반복해서 겪는 오류와 해결 방법을 정리합니다.
자주 발생하는 오류
1

메서드 이름 접미사 누락

이름이 ServerRpc/ClientRpc로 끝나지 않으면 어트리뷰트가 동작하지 않음. NGO가 경고 없이 무시.

2

NetworkBehaviour 미상속

MonoBehaviour만 상속한 클래스에서 RPC 선언 시 컴파일 또는 런타임 오류 발생.

3

스폰 전 RPC 호출

NetworkObject가 아직 스폰(Spawn)되지 않은 상태에서 호출하면 전송되지 않음.

4

클라이언트에서 ClientRpc 호출

서버만 호출 가능. 클라이언트에서 호출해도 조용히 무시됨. 디버깅이 어려움.

5

전달 불가 타입 사용

GameObject, MonoBehaviour 등을 파라미터로 쓰면 직렬화 오류 발생. NetworkObjectReference 사용 필요.

❌ 잘못된 패턴 — 클라이언트에서 ClientRpc 호출
// ❌ 클라이언트에서 호출 — 동작 안 함!
private void Update() {
    if (!IsServer) {
        ShowEffectClientRpc(pos); // 무시됨
    }
}
✔ 올바른 패턴
// ✔ 클라이언트 → ServerRpc → 서버에서 ClientRpc
[ServerRpc]
void RequestEffectServerRpc(Vector3 pos) {
    // 서버에서만 실행
    ShowEffectClientRpc(pos);
}
[ClientRpc]
void ShowEffectClientRpc(Vector3 pos) {
    SpawnParticle(pos);
}
🔑 성능 주의
매 프레임 RPC를 대량 호출하면 네트워크 과부하가 발생합니다. 위치·상태처럼 지속적으로 변하는 값은 RPC 대신 NetworkVariable이나 NetworkTransform을 사용하세요.

SUMMARY
핵심 요약 — RPC 완전 정리
RPC의 핵심 개념과 실전 사용 원칙을 한 장으로 정리합니다.
핵심 개념 총정리
01
RPC = 네트워크 이벤트 전달
NetworkVariable(상태)과 달리 일회성 이벤트를 전달. 결과 보장 없음.
02
[ServerRpc] — 클라이언트 → 서버
기본값 소유자만 호출 가능. 이름은 ServerRpc로 끝나야 함.
03
[ClientRpc] — 서버 → 클라이언트
서버만 호출 가능. 기본값 모든 클라이언트 수신. 이름은 ClientRpc로 끝나야 함.
04
직렬화 가능한 타입만 파라미터로 사용
GameObject 불가 → NetworkObjectReference 사용. 커스텀은 INetworkSerializable 구현.
05
황금 패턴: ServerRpc + NetworkVariable + ClientRpc
요청 → 검증·상태변경 → 결과 알림의 3단 구조가 가장 견고합니다.
빠른 참조 — 언제 무엇을 쓸까?
🗡️ 클라이언트 입력 전달 → ServerRpc
💬 채팅 요청 (누구나) → ServerRpc(RequireOwnership=false)
✨ 이펙트 동기 재생 → ClientRpc
🏆 게임 종료 알림 → ClientRpc
📬 특정 플레이어에게만 → ClientRpc + ClientRpcParams
❤️ 지속적으로 변하는 값 → NetworkVariable
